# Weed Varieties - Data Models

## Tables

### weed_varieties

| Column     | Type        | Constraints                                                                              |
|------------|-------------|------------------------------------------------------------------------------------------|
| id         | SERIAL      | PRIMARY KEY                                                                              |
| name       | TEXT        | NOT NULL                                                                                 |
| scent      | REAL        | NOT NULL, CHECK (`0 <= scent <= 10`)                                                     |
| flavor     | REAL        | NOT NULL, CHECK (`0 <= flavor <= 10`)                                                    |
| power      | REAL        | NOT NULL, CHECK (`0 <= power <= 10`)                                                     |
| quality    | REAL        | NOT NULL, CHECK (`0 <= quality <= 10`)                                                   |
| score      | REAL        | NOT NULL, GENERATED ALWAYS AS `(scent + flavor + power + quality) / 4.0` STORED          |
| price      | REAL        | NOT NULL                                                                                 |
| comments   | TEXT        | nullable                                                                                 |
| judge      | TEXT        | NOT NULL, DEFAULT `'Oscar'` — self-reported name of the rater                            |
| deleted_at | TIMESTAMPTZ | nullable — when set, row is hidden from API but kept for recovery                        |

**Indexes:**
- `idx_weed_varieties_alive` on `(score DESC, price ASC) WHERE deleted_at IS NULL` — backs the default list ordering, ignoring soft-deleted rows.

### weed_varieties_history

Append-only audit log written by the `weed_varieties_audit` trigger after every INSERT/UPDATE/DELETE on `weed_varieties`.

| Column           | Type        | Notes                                                       |
|------------------|-------------|-------------------------------------------------------------|
| history_id       | BIGSERIAL   | PRIMARY KEY                                                  |
| variety_id       | INT         | NOT NULL                                                     |
| op               | TEXT        | NOT NULL, one of `INSERT`/`UPDATE`/`DELETE`                  |
| changed_at       | TIMESTAMPTZ | NOT NULL, defaults to `now()`                                |
| name, scent, flavor, power, quality, score, price, comments, judge, deleted_at | … | snapshot of the row after the change (or before, for DELETE) |
| actor_ip         | TEXT        | client IP from `X-Forwarded-For` / `X-Real-IP` / `RemoteAddr`|
| actor_user_agent | TEXT        | `User-Agent` header                                          |
| actor_device_id  | TEXT        | `X-Device-ID` header (UUID generated by the web client and persisted in `localStorage`) |
| actor_token_kind | TEXT        | `semi` or `full` (or `backfill` for migration-seeded rows)   |

**Indexes:**
- `idx_weed_varieties_history_variety` on `(variety_id, changed_at DESC)`.

## Notes

- `score` is a stored generated column: it is recomputed by Postgres on every insert/update of the four sensory fields and cannot be written directly.
- `price` has no min/max — negative or zero values are allowed at the schema level.
- DELETE on `/varieties/{id}` is a **soft delete**: the API issues `UPDATE weed_varieties SET deleted_at = now() WHERE id = $1`. Hard recovery is `UPDATE weed_varieties SET deleted_at = NULL WHERE id = $1`.
- The audit trigger reads actor identifiers from `current_setting('app.actor_*', true)`. The Go repository sets these via `SELECT set_config(...)` inside the same transaction as each mutation; raw SQL changes outside the app will produce history rows with NULL actor fields.
- The table has no foreign keys.
